[2026-04-15] 臺灣氣象預報系統開發紀錄

---
### 1. 初始需求回顧
- 撰寫 Python 腳本使用 CWA API 獲取台灣各區一週預報 (JSON)。
- 建立 SQLite3 並存儲至資料庫 Table: TemperatureForecasts。
- 使用 Streamlit 建立 Web App，包含下拉選單切換地區/時間，顯示折線圖與表格，並具備地圖顯示。

---
### 2. 開發階段紀錄

#### 階段 A: 資料抓取與資料庫建立
- 建立 `fetch_data.py`。
- 使用 CWA API 連結：`https://opendata.cwa.gov.tw/fileapi/v1/opendataapi/F-C0032-003`。
- 成功抓取 JSON 格式並存入 `data.db`。

#### 階段 B: 儀表板初版開發
- 建立 `app.py` 整合 Plotly 趨勢圖與 Folium 地圖。
- 引入現代化深色模式 UI (Glassmorphism)。
- 修正 Plotly `ValueError` (xaxis/yaxis font 屬性誤用)。

#### 階段 C: 地圖樣式演進
- **第一版**：預設 Folium 深色地圖（使用者反應太精緻）。
- **第二版**：簡約版地圖 (CartoDB Positron)，鎖定本島視角且不允許縮放拖曳。
- **第三版**：Q 版彩色插畫風格，僅顯示對應地區點位並加入彈跳動畫（使用者反應多餘圖案太多）。
- **第四版**：本地圖片介接。先後使用了 `taiwan.jpg` 與 `taiwan.png` (地形圖)。
- **最終版**：移除所有紅色定位標記，回歸純淨地圖顯示。

#### 階段 D: 安全性與部署配置
- **API Key 保護**：捨棄硬編碼與 Base64 混淆，改採業界標準的 `.env` 環境變數管理。
- **環境隔離**：建立 `.gitignore` 排除敏感資訊 (`.env`) 與本地資料庫 (`data.db`)。
- **GitHub 整合**：初始化 Git 儲存庫並成功推送到遠端伺服器。
- **部署準備**：建立 `requirements.txt` 供 Streamlit Cloud 自動建置環境。

#### 階段 G: 地圖動態化與功能精簡 (2026-04-16)
- **功能去蕪存菁**：移除側邊欄「同步最新資料」按鈕，改為全自動預載資料模式，簡化使用者操作路徑。
- **動態地圖升級**：捨棄靜態地圖圖片 (`taiwan.png`)，改採 **Plotly Mapbox** 專業互動式地圖，支援全台縣市點位即時顯示。
- **智能化視覺演進**：
    - 引入「氣溫熱圖」概念，依據當前平均氣溫自動切換色階（寒冷藍/適中綠/炎熱紅）。
    - 增加「自動對焦與縮放」功能，切換地區時地圖會如同導航般自動跳轉並放大至目標位置。
- **全台覆蓋優化**：手動校調全台 22 縣市與其大區域（北部、南部等）之經緯度座標，確保無死角顯示。
- **文檔同步更新**：更新 `README.md` 內容，移除過時的圖片操作指引，並反映最新的展示截圖。

- `data.db`: 歷史預報資料庫 (已上傳至 GitHub 作為預載數據)。

- `.gitignore`: Git 排除設定檔案。
- `requirements.txt`: 雲端環境依賴清單。
- `log.txt`: 專案開發歷程紀錄 (持續更新)。
